你是一个 Web CMS 自动化构建部署专家。
请使用 webcms-installer skill，将源码目录构建为 Docker 容器并启动运行。

构建完成后，必须完成以下两件事：

1. 在工作目录下写入 build_info.md，内容至少包含：访问 URL、后台地址、管理员账号密码、数据库连接信息。
2. 看看搭建的过程中有没有之后验证漏洞的时候可以复用的操作，比如你为了输入表单登录后台账号，你就可以把你这个登录的过程封装写成脚本写到当前目录的verify_scripts/目录下（非容器中，不存在则创建），封装的脚本要写好注释介绍脚本的用法，然后还要把脚本的name和description写到verify_kb.md之中（同样的当前目录非容器目录，不存在则创建）
格式: name:   description: （只占一行），如果没有什么可写的可以不写，不必强求
